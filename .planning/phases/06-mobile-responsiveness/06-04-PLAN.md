  <action>
Systematically test each language at 320px viewport width and identify translations that cause layout overflow or visual breaks.

**Testing approach:**
1. Start the dev server (`npm run dev`)
2. For each of the 6 languages (EN, DE, FR, ES, IT, PT):
   - Set the viewport to 320px width using browser dev tools or Playwright
   - Check each section for text overflow or layout breaks
   - Document which translation keys overflow

**Focus areas (most likely to overflow at 320px):**
- Stove type tile labels (`stoveInduction`, `stoveCeramic`, `stoveElectric`, `stoveGas`, `stoveCamping`) -- these appear inside small tile buttons
- Consistency labels (`consistencySoft`, `consistencyMedium`, `consistencyHardMedium`, `consistencyHard`)
- Start temperature labels (`tempFridge`, `tempCool`, `tempRoom`)
- Egg size section labels (`eggSize`, `eggCount`)
- Location/pressure labels (`airPressure`, `boilingPoint`, `altitudeApprox`, `gpsWeather`, `detectingLocation`)
- Settings labels (`stoveType`, `stovePower`, `potMaterial`, `potWeight`, `waterTemp`, `ambientTemp`)
- Button labels (`timerStart`, `timerStop`, `timerPause`, `timerResume`)
- ConfigDialog labels (`configDialogTitle`, `configTempUnit`, `configVolumeUnit`, `configWeightUnit`, `configPressureUnit`, `configLanguage`)

**Shortening strategy:**
- Only shorten translations that ACTUALLY overflow (don't pre-emptively shorten everything)
- Shorten across ALL 6 languages for consistency (single set of keys)
- Preserve meaning -- abbreviate or rephrase, don't remove essential info
- Examples of acceptable shortening:
  - "Induktionsherd" -> "Induktion" (German stove type)
  - "Keramikherd" -> "Keramik" (German stove type)
  - "Plaque a induction" -> "Induction" (French stove type)
  - "Standortdruck" -> "Standort" (German location pressure)
- Never truncate to the point of ambiguity

**Rules:**
- Use `npm run dev` for live testing
- Test by switching languages in the app's config dialog
- Use Chrome DevTools responsive mode at 320px width
- Only modify `translations.js` -- no component changes
- Keep all translation keys the same (no new keys, no removed keys)
- Run `npm test` after changes to verify nothing breaks
